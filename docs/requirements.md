# Функциональные требования — mastroplan_bot

## Обзор

Telegram-бот `@mastroplan_bot` — персональный ассистент для управления задачами, календарём и продуктивностью. Бот доступен для ограниченного круга пользователей (whitelist по Telegram user_id), поддерживает голосовые команды и AI-парсинг намерений.

---

## 1. Пользователи и доступ

- Доступ только по whitelist (`ALLOWED_USER_IDS` в конфиге)
- При первом обращении пользователь автоматически регистрируется в БД
- Каждый пользователь имеет независимые данные: задачи, списки, настройки, токены
- Команда `/start` — приветствие и краткая инструкция
- Команда `/help` — список всех команд

---

## 2. Списки задач

### 2.1 Управление списками
- Пользователь создаёт произвольные списки задач: "Работа", "Дом", "Личное" и т.д.
- У каждого списка: название, эмодзи, цвет
- Команда `/lists` — просмотр всех списков с количеством задач
- Создание, переименование, удаление списков через inline-кнопки
- При регистрации создаются дефолтные списки: 💼 Работа, 🏠 Дом, 👤 Личное

### 2.2 Задачи
- Каждая задача принадлежит одному списку
- Поля задачи: название, список, приоритет (low/medium/high), дедлайн, статус
- Создание через свободный текст, голосовое сообщение или команду
- Просмотр: `/tasks` — все активные задачи, сгруппированные по спискам
- Фильтры: сегодня, просроченные, по списку, по приоритету
- Отметить выполненной, редактировать, переместить в другой список, удалить — через inline-кнопки
- Все события (создание, выполнение, перенос, удаление) записываются в `task_events` для аналитики

---

## 3. Календарь и напоминания

### 3.1 События
- Создание события через свободный текст или голосовое сообщение
- Поля: название, дата и время, напоминание (за N минут/часов до)
- Повторяющиеся события: ежедневно, еженедельно, ежемесячно
- Просмотр: `/calendar` — события на сегодня и ближайшие 7 дней

### 3.2 Google Calendar интеграция
- Подключение через `/connect_google` — OAuth2 flow
- После подключения все события создаются в Google Calendar пользователя
- Токены хранятся в БД в зашифрованном виде (Fernet), автоматически обновляются
- Отключение через `/disconnect_google`

### 3.3 Напоминания
- Отправка уведомления в Telegram в указанное время
- Напоминания без события в календаре (просто "напомни мне в 15:00 про звонок")
- APScheduler проверяет напоминания каждую минуту

---

## 4. Голосовые сообщения

- Бот принимает голосовые сообщения в любом месте диалога
- Аудио скачивается с серверов Telegram (формат OGG)
- Конвертируется в MP3 через ffmpeg
- Транскрибируется через OpenAI Whisper API
- Полученный текст обрабатывается так же как текстовое сообщение (через intent парсинг)
- Бот показывает транскрибированный текст перед выполнением действия

---

## 5. AI и парсинг намерений

### 5.1 Выбор модели
- Доступные модели: Claude (claude-sonnet-4-6) и GPT-4o
- Переключение через `/settings` или команду `/model`
- Настройка сохраняется per user

### 5.2 Парсинг намерений
Любое текстовое или голосовое сообщение проходит через intent parser:

**Поддерживаемые намерения:**
- `create_task` — создать задачу
- `create_event` — создать событие в календаре
- `create_reminder` — напоминание без события
- `list_tasks` — показать задачи (с возможными фильтрами)
- `list_events` — показать события календаря
- `complete_task` — отметить задачу выполненной
- `delete_task` — удалить задачу
- `update_task` — изменить задачу
- `get_briefing` — показать утренний брифинг
- `get_analytics` — показать аналитику
- `ai_chat` — свободный диалог с AI (нет конкретного действия)

**Формат ответа AI:**
```json
{
  "intents": [...],
  "confidence": 0.95,
  "clarification_needed": null
}
```

**Подтверждение перед выполнением:**
- Если `confidence < 0.8` — показать что понял бот и попросить подтвердить
- Если `clarification_needed != null` — задать уточняющий вопрос
- Деструктивные действия (delete) — всегда требуют подтверждения

### 5.3 Контекст диалога
- Последние 10 сообщений пользователя хранятся в `chat_history`
- Передаются в каждый запрос к AI для понимания контекста
- Пример: "перенеси на час позже" — AI понимает о каком событии речь из контекста

---

## 6. Автоматические рассылки

### 6.1 Утренний брифинг
Отправляется каждый день в настраиваемое время (по умолчанию 8:00).
Также доступен по команде `/morning` в любой момент.

Содержит:
1. **Просроченные задачи** (если есть) — выделены, требуют внимания
2. **События из Google Calendar** на сегодня
3. **Задачи на сегодня** — сгруппированы по спискам
4. **AI-комментарий** — короткий совет или оценка загруженности дня

### 6.2 Недельный план (понедельник утром)
Отправляется в понедельник в то же время что и утренний брифинг.

Содержит:
1. Задачи с дедлайном на текущую неделю — по спискам
2. События из Google Calendar на неделю
3. AI-комментарий: "На этой неделе X встреч и Y задач, приоритеты..."

### 6.3 Недельное саммари (воскресенье вечером, 20:00)
Также доступно по команде `/weekly` в любой момент.

Содержит:
1. **Статистика задач:** выполнено N из M, по каждому списку отдельно
2. **График продуктивности** по дням недели (текстовый, через символы)
3. **AI-инсайты:** паттерны, что западает, рекомендации на следующую неделю

---

## 7. Аналитика

Доступна через `/analytics` или через AI-запрос.

**Метрики:**
- Процент выполнения задач по спискам за выбранный период
- Динамика продуктивности по дням (сколько задач закрыто)
- Распределение задач по приоритетам
- Среднее время выполнения задачи
- Самые продуктивные дни недели

Данные берутся из таблицы `task_events`.

---

## 8. Настройки пользователя

Команда `/settings` открывает меню настроек:

- **Время брифинга** — когда присылать утренний план (по умолчанию 8:00)
- **Часовой пояс** — для корректного отображения времени
- **AI модель** — Claude или GPT-4o
- **Google Calendar** — статус подключения, подключить/отключить
- **Язык** — русский/английский (интерфейс бота)

---

## 9. UX и интерфейс

- Inline-кнопки для всех основных действий (не только команды)
- Команды работают на русском и английском
- Подтверждение перед деструктивными действиями
- Бот показывает транскрибированный текст голосового сообщения
- Понятные сообщения об ошибках
- `/cancel` — отмена любого текущего действия

---

## 10. Безопасность

- Все запросы проверяются на принадлежность к whitelist
- Google OAuth токены шифруются Fernet перед сохранением в БД
- Webhook URL содержит secret token, aiogram проверяет подпись каждого запроса
- Переменные окружения с секретами никогда не попадают в репозиторий
