# Функциональные требования — mastroplan_bot

## Обзор

Telegram-бот `@mastroplan_bot` — персональный ассистент для управления задачами, календарём и продуктивностью. Бот доступен для ограниченного круга пользователей (whitelist по Telegram user_id), поддерживает голосовые команды и AI-парсинг намерений.

---

## 1. Пользователи и доступ

- Доступ только по whitelist (`ALLOWED_USER_IDS` в конфиге)
- При первом обращении пользователь автоматически регистрируется в БД
- Каждый пользователь имеет независимые данные: задачи, списки, события, напоминания, настройки, токены
- Команда `/start` — приветствие, краткая инструкция, создание дефолтных списков задач
- Команда `/help` — список всех команд

---

## 2. Списки задач

### 2.1 Управление списками
- Пользователь создаёт произвольные списки задач: «Работа», «Дом», «Личное» и т.д.
- У каждого списка: название, эмодзи, цвет, порядок отображения
- Команда `/lists` — просмотр всех списков с количеством активных задач
- Создание, переименование, удаление списков через inline-кнопки (FSM)
- При удалении списка все его задачи удаляются каскадно — пользователь получает предупреждение перед подтверждением
- При регистрации создаются дефолтные списки: 💼 Работа, 🏠 Дом, 👤 Личное

### 2.2 Задачи
- Каждая задача принадлежит одному списку
- Поля задачи: название, список, приоритет (low/medium/high), дедлайн, статус (активна/выполнена)
- Создание через свободный текст, голосовое сообщение или команду
- Просмотр: `/tasks` — все активные задачи с inline-кнопками
- Фильтры через AI: сегодня, просроченные, по списку, по приоритету
- Действия через inline-кнопки: отметить выполненной, удалить (с подтверждением), переместить в другой список
- Все события (создание, выполнение, перенос, удаление, обновление) записываются в `task_events` для аналитики
- При неоднозначном поиске по названию (несколько совпадений) бот выводит список и просит уточнить

---

## 3. Календарь и напоминания

### 3.1 События календаря
- Создание события через свободный текст или голосовое сообщение
- Поля: название, дата и время начала, дата и время окончания (опционально), повтор
- Повторяющиеся события: ежедневно, еженедельно, ежемесячно
- Просмотр: `/calendar` — события на ближайшие 7 дней с inline-кнопками
- Удаление через inline-кнопку в детальном просмотре (синхронно удаляется из Google Calendar если подключён)

### 3.2 Напоминания
Единая сущность для двух сценариев:

**Standalone-напоминание** — не привязано к событию:
> «Напомни мне через 10 минут выключить курицу»

**Напоминание к событию** — создаётся при создании события, привязано к нему через `event_id`:
> «Завтра стрижка в 18:00, напомни за час до неё»
> «Создай встречу в пятницу в 15:00 и напомни за 30 минут и за 5 минут»

- К одному событию можно привязать **несколько напоминаний** с разными смещениями
- При удалении события все его напоминания удаляются каскадно
- Просмотр: `/reminders` — все активные напоминания с inline-кнопками; привязанные к событию помечены 📅
- Удаление через inline-кнопку (с подтверждением)
- Управление через AI: создать, показать список, удалить, изменить время/название
- APScheduler проверяет напоминания каждую минуту
- Повторяющиеся напоминания: после срабатывания `remind_at` продвигается вперёд до следующего момента после `now` — защита от многократного срабатывания при даунтайме

### 3.3 Google Calendar интеграция
- Подключение через `/connect_google` — OAuth2 flow
- После подключения события создаются в Google Calendar пользователя
- Токены хранятся в БД в зашифрованном виде (Fernet), автоматически обновляются
- Отключение через `/disconnect_google`
- Синхронизация односторонняя: бот → Google. События, созданные напрямую в Google Calendar, в боте не отображаются

---

## 4. Голосовые сообщения

- Бот принимает голосовые сообщения в любом месте диалога
- Аудио скачивается с серверов Telegram (формат OGG)
- Конвертируется в MP3 через ffmpeg
- Транскрибируется через OpenAI Whisper API
- Полученный текст обрабатывается так же как текстовое сообщение (через intent-парсинг)
- Бот показывает транскрибированный текст перед выполнением действия

---

## 5. AI и парсинг намерений

### 5.1 Выбор модели
- Доступные модели: Claude (`claude-sonnet-4-6`) и GPT-4o
- Переключение через `/settings` → «AI модель» или команду `/model`
- Настройка сохраняется per user

### 5.2 Парсинг намерений
Любое текстовое или голосовое сообщение проходит через intent-парсер.

**Поддерживаемые намерения:**

| Intent | Описание |
|---|---|
| `create_task` | Создать задачу |
| `create_event` | Создать событие в календаре (с опциональными напоминаниями) |
| `create_reminder` | Standalone-напоминание без события |
| `list_tasks` | Показать задачи (с фильтрами) |
| `list_events` | Показать события календаря |
| `list_reminders` | Показать все активные напоминания |
| `complete_task` | Отметить задачу выполненной |
| `delete_task` | Удалить задачу |
| `update_task` | Изменить задачу |
| `delete_reminder` | Удалить напоминание |
| `update_reminder` | Изменить время или название напоминания |
| `get_briefing` | Показать утренний брифинг |
| `get_analytics` | Показать аналитику (за неделю или месяц) |
| `ai_chat` | Свободный диалог с AI |

**Формат ответа AI:**
```json
{
  "intents": [...],
  "confidence": 0.95,
  "clarification_needed": null
}
```

**Логика подтверждения:**
- `confidence < 0.8` — показать что понял бот, попросить подтвердить
- `clarification_needed != null` — задать уточняющий вопрос
- Деструктивные действия (`delete_task`, `delete_reminder`) — всегда требуют подтверждения

**Мультиинтент:** одно сообщение может содержать несколько намерений:
> «Добавь задачу позвонить врачу и напомни мне об этом в 14:00»

### 5.3 Контекст диалога
- Последние 10 сообщений пользователя хранятся в `chat_history`
- Передаются в каждый запрос к AI (intent-парсер и свободный диалог)
- Пример: «перенеси на час позже» — AI понимает о каком событии речь из контекста

## 6. Автоматические рассылки

### 6.1 Утренний брифинг
Отправляется каждый день в настраиваемое время (по умолчанию 8:00 **в часовом поясе пользователя**).
Также доступен по команде `/morning` в любой момент.

Содержит:
1. **Просроченные задачи** (если есть) — выделены, требуют внимания
2. **События из локального календаря** на сегодня
3. **Задачи на сегодня** — по дедлайну
4. **AI-комментарий** — короткий совет или оценка загруженности дня

### 6.2 Недельный план (понедельник утром)
Отправляется в понедельник вместе с утренним брифингом (в то же настроенное время).

Содержит:
1. Задачи с дедлайном на текущую неделю — по спискам
2. События из локального календаря на неделю
3. AI-комментарий

### 6.3 Недельное саммари (воскресенье, 20:00 **в часовом поясе пользователя**)
Также доступно по команде `/weekly` в любой момент.

Содержит:
1. **Статистика задач:** создано N, выполнено M, по каждому списку отдельно
2. **График продуктивности** по дням недели (текстовый, символьный bar chart)
3. **AI-инсайты:** паттерны, рекомендации на следующую неделю

---

## 7. Аналитика

Доступна через `/analytics` или AI-запросом «покажи аналитику за месяц».

**Периоды:** неделя (7 дней), месяц (30 дней)

**Метрики:**
- Количество созданных и выполненных задач за период
- Процент выполнения по каждому списку
- Динамика по дням (неделя) или по неделям (месяц) — bar chart символами
- AI-инсайты: паттерны, самый продуктивный день, рекомендации

Данные берутся из таблицы `task_events`.

---

## 8. Настройки пользователя

Команда `/settings` открывает меню настроек:

| Настройка | Описание | По умолчанию |
|---|---|---|
| Время брифинга | Когда присылать утренний план | 08:00 |
| Часовой пояс | Для корректного отображения времени | Europe/Moscow |
| AI модель | Claude или GPT-4o | Claude |
| Google Calendar | Статус подключения, подключить/отключить | — |

## 9. UX и интерфейс

- Inline-кнопки для всех основных действий (не только текстовые команды)
- Подтверждение перед деструктивными действиями (удаление задачи, напоминания, списка)
- При неоднозначном поиске (несколько совпадений по названию) — бот выводит список и просит уточнить
- Бот показывает транскрибированный текст голосового сообщения перед выполнением
- Понятные сообщения об ошибках
- `/cancel` — отмена любого текущего FSM-действия

**Команды бота:**
```
/tasks          — активные задачи
/lists          — управление списками
/calendar       — события на 7 дней
/reminders      — активные напоминания
/morning        — утренний брифинг
/weekly         — недельное саммари
/analytics      — статистика
/settings       — настройки
/model          — выбор AI модели
/connect_google — подключить Google Calendar
/disconnect_google — отключить Google Calendar
/help           — справка
/cancel         — отмена действия
```

---

## 10. Безопасность

- Все запросы проверяются на принадлежность к whitelist (middleware на каждый update)
- Google OAuth токены шифруются Fernet (`ENCRYPTION_KEY`) перед сохранением в БД
- Webhook URL содержит secret token, aiogram проверяет подпись каждого запроса
- Переменные окружения с секретами никогда не попадают в репозиторий (`.gitignore`, gitleaks в CI)
- PostgreSQL и бот не доступны снаружи Docker-сети, наружу торчит только nginx на 80/443
